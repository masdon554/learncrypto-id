PROOF OF WORK (PoW)

  1. article
  2. basicz
  3. proof of work

The block chain is managed collaboratively by anonymous peers in the network, so Bitcoin requires that each block prove a significant amount of work to ensure that the nodes that want to modify the previous block have to work harder than honest counterparts who just want to add new blocks to the blockchain. With nodes chaining together blocks makes it impossible to change the transactions included in any block without modifying all subsequent blocks. As a result, the cost of modifying a particular block increases with each new block added to the blockchain, magnifying the proof-of-work effect.

The proof of work used in Bitcoin takes advantage of the seemingly random nature of cryptographic hashes. A good algorithm will convert arbitrary data into seemingly random numbers. To prove that you did some extra work to create the block, you must create a hash of the block header that does not exceed a certain value. This results in repeated block hash formation until a given hash produces a number below the target threshold. Bitcoin assumes a linear probability that the lower it makes the target threshold, the more hashes (on average) it will need to attempt.

Because each block header must have a value below the target threshold, and because each block is connected to the block that precedes it, it requires (on average) hashing power to propagate such a modified block issued throughout the Bitcoin network between the time the original block was created and the current time. The block header provides several fields that are easy to modify, such as a custom nonce field, so that obtaining a new hash does not require waiting for a new transaction.
